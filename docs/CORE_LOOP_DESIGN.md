## 两个智能体的职责

### Plan 智能体 (规划者)

**输入**：
- MetaDesign（元设计）
- 当前代码库状态
- 历史 Loop 记录

**输出**：
- 决策卡片（Cards）
  - 目标（objective）
  - 方法（approach）
  - 预期收益（benefit）
  - 预期成本（cost）
  - 风险（risk）
  - 预测置信度（confidence）

**权限**：
- ✓ 读取代码
- ✓ 分析代码
- ✓ 生成卡片
- ✗ 修改代码
- ✗ 执行命令

### Build 智能体 (执行者)

**输入**：
- 被接受的卡片
- 执行计划（Plan）
- 约束条件

**输出**：
- 代码修改
- Git 提交
- 执行结果

**权限**：
- ✓ 读取代码
- ✓ 修改代码
- ✓ 执行命令
- ✓ 创建 Git 提交

## 人类的作用

### 一次性动作：元设计

人类在项目开始时定义：
1. **Core Value** - 项目要达到的核心价值
2. **Anti Value** - 项目绝对不能变成的样子
3. **Requirements** - 元需求列表
4. **Constraints** - 硬约束
5. **Eval Factors** - 评估因子

### 循环动作：选择卡片

每轮 Loop 中，人类只需要：
1. 查看 Plan 智能体生成的卡片
2. 选择接受或拒绝
3. （可选）添加拒绝原因

系统自动完成：
- Plan 智能体分析代码并生成卡片
- Build 智能体执行被接受的卡片
- 评估执行结果
- 更新 MetaDesign 状态

## 优化方向

### 1. 简化人类操作

当前问题：
- 人类需要手动输入 `/meta` 开始 Loop
- 人类需要手动输入 `/meta-decide` 审查卡片
- 人类需要手动输入 `/meta-execute` 执行

优化方案：
- 自动检测项目状态
- 自动开始 Loop
- 自动展示卡片
- 人类只需点击接受/拒绝

### 2. 智能体自动切换

当前问题：
- Plan 和 Build 的切换需要手动触发
- 没有自动化的循环机制

优化方案：
```
while (true) {
  // Plan 阶段
  planAgent.analyze()
  cards = planAgent.generateCards()
  
  // 人类决策（等待）
  decisions = waitForHumanDecision(cards)
  
  // Build 阶段
  acceptedCards = filterAccepted(decisions)
  for (card of acceptedCards) {
    buildAgent.execute(card)
  }
  
  // 评估阶段
  evaluation = evaluate()
  updateMetaDesign(evaluation)
  
  // 自动开始下一轮
}
```

### 3. Dashboard 作为主要界面

当前问题：
- TUI 是主要界面，但有 bug
- Dashboard 功能不完整

优化方案：
- Dashboard 成为主要交互界面
- 实时显示 Plan 智能体的分析过程
- 一键接受/拒绝卡片
- 实时显示 Build 智能体的执行进度
- 自动开始下一轮 Loop

## 实现计划

### Phase 1: 核心循环自动化

1. 实现 `LoopOrchestrator` 类
   - 管理 Plan 和 Build 智能体的切换
   - 自动检测循环状态
   - 自动开始下一轮

2. 实现 `PlanAgent` 封装
   - 分析代码
   - 生成卡片
   - 输出到 Dashboard

3. 实现 `BuildAgent` 封装
   - 执行被接受的卡片
   - 自动提交代码
   - 报告执行结果

### Phase 2: Dashboard 增强

1. 实时显示 Plan 分析过程
2. 卡片一键接受/拒绝
3. Build 执行进度条
4. 评估结果可视化

### Phase 3: 智能优化

1. 根据历史数据优化卡片生成
2. 自动调整评估因子权重
3. 自动解锁 Negative
