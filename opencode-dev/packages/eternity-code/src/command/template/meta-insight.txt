Write a design insight based on recent development experience.

An insight is not a requirement or a task - it's the reasoning behind "why we designed it this way".

Good insights explain:
1. The decision that was made
2. Why it was made (constraints, trade-offs, goals)
3. What implications it has for future development

Categories:
- architecture: System design decisions
- product: User-facing feature decisions
- process: Development workflow decisions
- technical: Implementation details and patterns

Output format:
---INSIGHT START---
title: （insight title）
source: （where this insight came from）
category: architecture / product / process / technical
insight: |
  （the core insight - why this decision was made）
implications:
  - （implication 1）
  - （implication 2）
related:
  - （related insight or document）
---INSIGHT END---
